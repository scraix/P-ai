> 把 LLM 变成真正会干活的桌面 AI 指挥部。  
> 它不只是聊天窗口，而是一个能管理自己、管理手下、管理技能、管理工作空间，并长期持续执行任务的桌面 AI 操作系统。

Easy Call AI 的目标，从来都不是“再做一个 AI 聊天网页”。

它要做的是另一件更野心勃勃的事：

- 让 AI 可以被一键召唤到桌面边缘
- 让 AI 拥有稳定人格、稳定部门、稳定技能、稳定 MCP、稳定工作空间
- 让 AI 不只回答问题，还能接任务、追任务、委派任务、长期协作
- 让 AI 的记忆不再昂贵、短命、脆弱，而是可持续、可追踪、可归档

如果你想要的是“一个会说话的网页”，这里不是重点。  
如果你想要的是“一个真正能持续工作的桌面 AI 系统”，这里就是。

## 现在它是什么

Easy Call AI 现在更接近下面这几个东西的混合体：

- 桌面 AI 浮窗
- 长期运行的任务系统
- 可委派的部门 / 工具人系统
- Skill + MCP 工作区
- 低成本持续记忆系统
- 可自动归档和整理上下文的会话系统

一句话说完：

> 这是一个可召唤、可委派、可执行、可积累的桌面 AI 工作中枢。

## 为什么它和普通 AI 工具不一样

很多 AI 产品的问题，不是模型不够强，而是系统设计太原始：

| 常见问题 | 后果 |
|------|------|
| 只有聊天，没有任务 | AI 只能回答，不能持续推进工作 |
| 只有一个身份，没有组织 | AI 无法分工，所有事都挤在一个脑袋里 |
| 没有稳定工作空间 | 每一轮都像失忆重开 |
| 记忆昂贵且混乱 | 越聊越重，越聊越脆弱 |
| 工具和能力边界不清 | 容易失控，也难以复用 |

Easy Call AI 的设计方向相反：

- 让 LLM 有“身份”
- 让 LLM 有“部门”
- 让 LLM 有“手下”
- 让 LLM 有“技能”
- 让 LLM 有“工具边界”
- 让 LLM 有“长期工作空间”
- 让 LLM 有“低成本记忆”

这不是让 AI 更像聊天机器人，而是让 AI 更像一个真正活着的工作系统。

## 核心能力

### 1. 一键召唤的桌面 AI
- 全局热键呼出 / 隐藏对话窗口
- 托盘常驻，随时打开配置、对话、归档
- 不打断当前工作流，直接在桌面侧边协作

### 2. 多模型、多供应商、多协议统一运行
- 可管理多套 API 配置
- 支持不同模型承担不同职责
- 后端统一处理流式回复、工具调用、上下文与归档链路

### 3. 任务系统，而不是只会聊天
- AI 可以创建、维护、完成任务
- 支持到点触发、周期触发、持续追踪
- 有任务看板、任务详情、当前追踪任务
- 可以把“一个月的长期工作”拆成阶段推进，而不是一轮对话就忘掉

### 4. 部门与工具人委托系统
- 主助理可以把工作委派给下属部门
- 部门可以继续向下分工
- 后台执行，不阻塞前台聊天
- 支持委托回流、汇报、独立工作链

这意味着 AI 不再是“一个人硬做所有事”，而是开始具备组织能力。

### 5. Skill 工作区
- 内置预设 skill
- 支持在工作区中安装、编写、刷新 skill
- Skill 页面可直接查看 `SKILL.md` 描述与正文
- 支持从公开来源迁移能力模板，再按本项目规范落地

### 6. MCP 工具生态
- 支持 MCP 工具接入
- 能按人格 / 部门边界控制工具可用性
- 让 AI 的工具能力不再是混乱拼接，而是可治理的运行时能力

### 7. 私有部门与私有人格
- 主配置之外，AI 还可以拥有自己的私有组织空间
- 私有部门、私有人格、私有 skill 可以从工作区刷新进入运行时
- 主助理与私有组织并存，不必把所有配置都塞进一份全局文件

### 8. 高效且省成本的记忆机制
- 主动回忆
- 记忆分类
- 私有记忆
- 长对话自动归档
- 上下文整理与压缩
- 后端统一计算上下文占用

目标不是“无限塞上下文”，而是让 AI 长期活着，还尽量别烧钱。

## 一个典型工作流会是什么样

### 临时提问
1. 热键呼出对话窗
2. 直接提问、贴图、截图
3. AI 立刻回答或调用工具
4. 结束后收起，继续工作

### 长期任务追踪
1. AI 创建任务
2. 设定触发时间、周期或持续推进目标
3. 到点自动提醒
4. 每轮处理后记录状态，而不是简单“做完就消失”

### 部门协作
1. 主助理接到任务
2. 委派给对应部门 / 工具人
3. 子部门独立执行
4. 结果回流到上级
5. 最终由主助理对用户负责

### 工作区进化
1. AI 在自己的工作区维护 skill、私有人格、私有部门
2. 刷新后即时进入运行时
3. 组织结构和能力边界随着工作长期演化

## Arch Linux 安装（yay）

如果你在 Arch Linux / Manjaro 上，希望从本项目仓库直接安装：

```bash
git clone https://github.com/kawayiYokami/Easy-call-ai.git
cd Easy-call-ai/packaging/arch
chmod +x install-with-yay.sh
./install-with-yay.sh
```

脚本会自动更新 `PKGBUILD` 的 `pkgver` 到最新 release，并执行 `yay -Bi` 构建与安装。

安装后主要文件位置：

- 可执行文件：`/usr/bin/easy-call-ai`
- 桌面启动项：`/usr/share/applications/easy-call-ai.desktop`
- 图标：`/usr/share/pixmaps/easy-call-ai.png`
- 默认数据目录：`~/.config/easy-call-ai/`

## 隐私与数据

- API Key 默认保存在本机
- 对话、归档、任务、记忆、工作区数据默认保存在本地
- 你可以自行管理、导出、清理这些数据
