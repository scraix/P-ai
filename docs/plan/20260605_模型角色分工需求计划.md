# 模型角色分工需求计划

## 目标

减少用户配置模型时的决策成本。用户只需要在「模型分工」里设置专家模型和快速模型，其他位置默认使用明确的角色选项，不再出现“无”或隐式跟随语义。

## 口径

- 「专家模型」是主要对话和专家任务默认使用的角色模型。
- 「快速模型」是工具审查、联系人秘书等轻量固定流程使用的角色模型。
- 可选模型的普通下拉默认选「专家模型」。
- 角色选项必须显式显示为「专家模型」「快速模型」，不使用「无」「none」「pro」等表达。
- 具体调用前统一把角色解析为「模型分工」中配置的真实模型。

## 实现范围

- 新增前后端统一角色值：`role:expert`、`role:quick`。
- 部门默认驱动模型改为「专家模型」角色。
- LLM 私域/部门配置允许引用角色值，并在运行时解析。
- 会话单次/首选模型若传入角色值，运行时解析到对应真实模型。
- 欢迎页在专家模型或快速模型未设置时提示配置。
- 快速设置创建第一个模型后，同步设置专家模型和快速模型为该模型。

## 验证

- 前端类型检查：`pnpm typecheck`
- 后端编译检查：`cd src-tauri && cargo check`
