# LLM 本地图片发送计划

## 背景

当前聊天消息已经支持用户上传图片、历史图片懒加载、表情包分段渲染，以及远程联系人发送表情包。用户希望新增一套让 LLM 在回复正文里引用本地图片的语法：图片不归档、不复制原图，只在本地会话和远程联系人发送时按路径读取；如果文件被删除或移动，允许显示裂图或不可用状态。

该需求跨后端文本解析、前端消息渲染、图片预览操作、远程 IM 发送和测试验证，先按计划确认后再实现。

## 目标

- LLM 可以在回复正文中用标准 Markdown 图片语法发送本地图片。
- 本地聊天中显示长边不超过 1080 的缩略图，单击后读取并显示原图。
- 原图预览中支持下载原图和复制图片到剪贴板。
- 不复制、不持久化原图到 `media/`；图片路径失效时允许显示不可用占位。
- 远程联系人回复也解析同一套语法：长边 2160 以内发送原图，超过 2160 时压缩后发送。
- 远程发送方法复用表情包的 content item 形态：`type=image`、`mime`、`name`、`bytesBase64`。
- 实现完成后先自我 review 并修复明显问题，再给用户实测清单。

## 非目标

- 不做图片生成模型接入。
- 不把本地图片复制到 `media/` 或会话存储目录。
- 不让 Markdown renderer 直接加载 `file://` 或本地绝对路径。
- 不保证历史会话里的本地路径长期可用。
- 不把远程 URL、`data:` 当作本地图片功能输入；`file://` 只做兼容解析，不作为推荐输出。

## 语法设计

正式语法使用标准 Markdown 图片语法：

```text
![结果图](E:\path\to\result.png)
![结果图](outputs/result.png)
```

规则：

- `![说明](路径)` 是唯一正式语法，说明文字用于 `alt`、下载默认名和占位提示。
- Windows 盘符里的 `:` 不需要转义；路径中的 `)` 建议用 Markdown 转义或尖括号包裹。
- 相对路径按当前配置工作区根目录解析，绝对路径按原路径解析；口径与现有 `contact_send_files` / meme 源文件路径保持一致。
- 支持格式先限定为常见 raster：png、jpg/jpeg、webp、gif、bmp；svg/heic/heif 暂不进入本功能。

## 数据模型

新增统一的消息内联展示段，优先写入 `providerMeta.inlineSegments`：

```ts
type InlineMessageSegment =
  | { type: "text"; text: string }
  | { type: "meme"; name: string; category: string; mime: string; relativePath: string; bytesBase64: string }
  | {
      type: "localImage";
      path: string;
      fileName: string;
      mime: string;
      alt?: string;
      width?: number;
      height?: number;
    };
```

说明：

- `parts[].text` 保留 LLM 原始文本，便于调试和上下文回放。
- 前端展示优先使用 `providerMeta.inlineSegments`，没有该字段时继续兼容旧的 `providerMeta.memeSegments`。
- 本地图片段只保存路径和元信息，不保存 `bytesBase64`。
- 如果解析失败，本地聊天可显示占位；远程发送倾向于让工具调用失败，避免误发漏图。

## 后端实现

### 1. 本地图片解析与元信息

新增本地图片解析 helper，建议放在：

- `src-tauri/src/features/chat/model_runtime/tools_and_builtin/builtin_local_image.rs`

职责：

- 解析 Markdown 图片语法 `![alt](path)`，输出 text/localImage 分段。
- 解析路径：绝对路径直接使用，相对路径基于 `configured_workspace_root_path(state)`，失败时回退 `state.llm_workspace_path`。
- 校验文件存在且是普通文件。
- 用文件扩展名和实际字节识别 mime，避免扩展名与内容严重不一致。
- 读取图片尺寸，写入 `width/height`。
- 不复制原图。

### 2. 助手消息落库

在助手最终回复落库前，当前 meme 会调用：

- `resolve_text_to_persisted_meme_segments(...)`
- `persist_meme_segments_into_provider_meta(...)`

计划改为统一解析：

- 先解析 Markdown 本地图片。
- 对 text 段继续解析 meme token，保证 `:meme:` 和 `![alt](path)` 可以混用并保持顺序。
- 生成 `providerMeta.inlineSegments`。
- 旧 `memeSegments` 可继续写入或保留兼容，但前端以 `inlineSegments` 优先。

涉及文件：

- `src-tauri/src/features/chat/model_runtime/tools_and_builtin.rs`
- `src-tauri/src/features/chat/model_runtime/tools_and_builtin/builtin_meme.rs`
- `src-tauri/src/features/chat/model_runtime/tools_and_builtin/builtin_local_image.rs`
- `src-tauri/src/features/system/commands/chat_and_runtime/core_send_inner.rs`

### 3. 本地图片读取命令

新增 Tauri 命令：

- `read_local_chat_image_thumbnail`
  - 输入：`path`、`maxEdge`，默认 1080。
  - 输出：`dataUrl`、`mime`、`width`、`height`、`originalWidth`、`originalHeight`。
  - 行为：长边超过 1080 时只在内存中缩小并编码 WebP；长边 1080 内可直接返回原图 data URL。
  - 缓存：优先做内存 LRU，key 使用 path + file size + modified time + maxEdge，不落盘。

- `read_local_chat_image_original`
  - 输入：`path`。
  - 输出：原图 `dataUrl`、`mime`、`width`、`height`。
  - 只在单击预览时调用。

- `copy_local_chat_image_to_clipboard`
  - 输入：`path`。
  - 行为：gif/webp 按原文件写入剪贴板文件列表以保留动图；png/jpg/bmp 按位图写入系统剪贴板。

- `save_local_chat_image_as`
  - 输入：`path`。
  - 行为：弹出保存对话框，将原文件复制到用户选择位置。

涉及文件：

- `src-tauri/src/features/system/commands/config_and_persona/persona_and_chat_settings.rs` 或拆到 chat/runtime commands 中更合适的位置。
- `src-tauri/src/main.rs` 注册命令。
- 可复用 `media_mime_from_path`、`image_mime_from_bytes`、`arboard`、`image`、`webp`。

### 4. 远程联系人发送

当前 `remote_im_build_text_content_items` 会解析 meme 并转成 content items。计划扩展为：

- 解析同一套 `inlineSegments`。
- text 段输出 `{ type: "text", text }`。
- meme 段继续走现有表情包转换。
- localImage 段读取本地文件：
  - 长边 <= 2160：发送原图 bytes，保留原 mime 和文件名。
  - 长边 > 2160：压缩到长边 <= 2160，编码 WebP，文件名改为原 stem + `.webp`。
- 输出仍为 `{ type: "image", mime, name, bytesBase64 }`，交给现有平台 SDK 发送。
- 本地图片不存在、格式不支持、读取失败时，`contact_reply` 应失败并带明确错误，避免远程联系人收到缺图消息。

涉及文件：

- `src-tauri/src/features/chat/model_runtime/tools_and_builtin/builtin_remote_im.rs`
- `src-tauri/src/features/chat/model_runtime/tools_and_builtin/builtin_meme.rs`
- 新增本地图片 helper。

## 前端实现

### 1. 类型与语义投影

新增/扩展类型：

- `src/types/app.ts` 增加 `InlineMessageSegment`。
- `ChatMessageBlock` 增加 `inlineSegments?: InlineMessageSegment[]`。
- `src/utils/chat-message-semantics.ts` 新增 `resolveInlineSegments`：
  - 优先读取 `providerMeta.inlineSegments`。
  - 兼容旧 `providerMeta.memeSegments`。

### 2. 消息渲染

调整 `ChatMessageItem.vue`：

- 当前 `hasRenderableMemeSegments` 扩展为 `hasRenderableInlineSegments`。
- text 段继续用 Markdown 渲染。
- meme 段沿用现有 inline meme 样式。
- localImage 段显示缩略图：
  - 首次进入视口或渲染时调用 `read_local_chat_image_thumbnail(maxEdge=1080)`。
  - 显示固定最大尺寸和占位，避免布局跳动。
  - 单击调用原图读取并打开预览。
  - 文件缺失或读取失败时显示本地图片不可用占位。

### 3. 原图预览操作

调整 `ChatImagePreviewDialog.vue` 和相关 composable：

- 弹窗打开本地图片时读取原图。
- 工具栏增加图标按钮：复制图片、下载图片、缩小、放大、重置。
- 复制调用 `copy_local_chat_image_to_clipboard`。
- 下载调用 `save_local_chat_image_as`。
- 现有用户上传/历史 `dataUrl` 图片预览保持原行为；只有 localImage 才显示本地原图下载/复制入口。

## 提示词与工具描述

- 在合适的 prompt rule 中说明：如果要把本地图片贴到聊天里，输出标准 Markdown 图片语法 `![说明](path)`，不要输出 `file://`。
- 规则应强调：只能引用已经存在的本地图片文件，不要编造路径。

## 测试与验证

开发阶段只跑本次影响范围的必要测试：

- Rust parser 单测：普通文本、单图、多图、meme 混排、相对路径、绝对路径、无效路径。
- Rust 图片处理单测：1080 缩略图长边限制、2160 远程压缩长边限制、小图原图发送、坏图报错。
- Rust 远程 content item 单测：本地图片 token 转成 `type=image`，并保持 text/image 顺序。
- 前端类型检查：`pnpm typecheck`。
- 后端检查：`cd src-tauri && cargo check`。
- 必要时跑相关 Rust 单测，不跑无关全量测试。

## 自我 Review 清单

实现完成后先自查并修复问题，再交付给用户：

- 消息正文、meme、本地图片混排顺序是否正确。
- 本地图片不存在时，聊天窗口是否占位，远程发送是否失败并有明确错误。
- 缩略图是否长边 <= 1080，原图预览是否读取原图而不是缩略图。
- 下载是否复制原文件，复制到剪贴板是否可用。
- 远程发送长边 <= 2160 是否原图，> 2160 是否压缩后发送。
- 是否没有把原图写入 `media/` 或消息 `parts[].image.bytesBase64`。
- 是否没有让 Markdown renderer 直接访问本地路径。
- 是否没有引入无关格式化或大范围重构。

## 用户实测清单草案

实现和自我 review 完成后给用户最终版实测清单，预计包含：

- 本地聊天发送 `![说明](绝对路径)`，应显示缩略图。
- 本地聊天发送 `![说明](相对路径)`，应按工作区解析并显示缩略图。
- 删除图片后刷新或切换会话，应显示不可用占位。
- 单击缩略图，应打开原图预览。
- 原图预览复制图片到剪贴板，应能粘贴到微信或画图等应用。
- 原图预览下载，应保存原始文件。
- 远程联系人回复包含小图 token，应收到原图。
- 远程联系人回复包含超大图 token，应收到压缩图。
- 文本、`:meme:`、`![alt](path)` 混排时顺序应正确。

## 待确认

- 本计划默认允许绝对路径，行为与现有 `contact_send_files` 一致；如果后续需要收紧，可改为只允许工作区内路径。
- 动图复制到剪贴板优先保留原文件；远程发送小动图会尽量保留原图，超 2160 压缩时可能退化为静态 WebP。
