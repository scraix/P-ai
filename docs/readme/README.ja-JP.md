> LLM を本当に仕事ができるデスクトップ AI 指揮中枢へ変える。  
> これは単なるチャットウィンドウではなく、自分自身、部下、スキル、ワークスペースを管理し、長期にわたって仕事を進め続けられるデスクトップ AI オペレーティングシステムです。

Easy Call AI の目的は、「また 1 つの AI チャットサイトを作ること」ではありません。

目指しているのは、もっと野心的なことです。

- 1 つのホットキーで AI をデスクトップの端に召喚する
- AI に安定した人格、部門、スキル、MCP、ワークスペースを持たせる
- AI を単なる応答装置ではなく、タスク受領、追跡、委任、長期協働ができる存在にする
- 記憶を高コストで脆いものではなく、継続可能で追跡可能、アーカイブ可能なものにする

ただ会話する Web ページが欲しいなら、ここは本質ではありません。  
長く働き続けられるデスクトップ AI システムが欲しいなら、ここがその場所です。

## 今これは何なのか

Easy Call AI は、次のものを組み合わせた存在に近いです。

- デスクトップ AI オーバーレイ
- 長期稼働するタスクシステム
- 部門 / 工具人による委任システム
- Skill + MCP ワークスペース
- 低コストな持続記憶システム
- 自動アーカイブと文脈整理を備えた会話システム

一言で言えば：

> 召喚でき、委任でき、実行でき、蓄積できるデスクトップ AI 作業中枢です。

## なぜ普通の AI ツールと違うのか

多くの AI 製品の限界は、モデル性能よりもシステム設計の原始性にあります。

| よくある問題 | その結果 |
|------|------|
| チャットしかなく、タスクがない | 答えられても、仕事を前に進め続けられない |
| 1 つの人格しかなく、組織がない | すべてを 1 人の助手に押し込むことになる |
| 安定したワークスペースがない | 毎回少しずつリセットされたようになる |
| 記憶が高コストで混乱しやすい | 文脈が重く、脆くなる |
| ツール境界が曖昧 | 制御もしにくく、再利用もしにくい |

Easy Call AI は逆方向を目指します。

- LLM に「身分」を与える
- LLM に「部門」を与える
- LLM に「部下」を与える
- LLM に「スキル」を与える
- LLM に「ツール境界」を与える
- LLM に「長期ワークスペース」を与える
- LLM に「低コストな持続記憶」を与える

目標は、AI をチャットボットらしくすることではありません。  
AI を本当に働き続けるシステムにすることです。

## コア機能

### 1. ワンキー召喚のデスクトップ AI
- グローバルホットキーで即座に表示 / 非表示
- システムトレイ常駐
- いまの作業を壊さず、画面端で協働

### 2. 統一されたマルチモデル実行基盤
- 複数のプロバイダとモデル設定を管理
- 役割ごとに異なるモデルを割り当て
- ストリーミング、ツール呼び出し、文脈、アーカイブを 1 本のバックエンド経路で統一

### 3. 本物のタスクシステム
- AI 自身がタスクを作成、維持、完了できる
- 指定時刻、定期実行、継続追跡に対応
- タスクボード、詳細、現在追跡中タスクを持つ
- 1 か月単位の仕事でも段階的に前進できる

### 4. 部門と工具人の委任
- 主助理は仕事を下位部門へ委任できる
- 部門はさらに下へ分配できる
- バックグラウンド実行で前景チャットを塞がない
- 結果は上位へ流れて戻る

つまり、AI は 1 人ですべてを抱え込む存在ではなく、組織として動き始めます。

### 5. Skill ワークスペース
- 組み込み preset skill
- ワークスペース内で skill を追加、作成、更新、再読込
- Skill ページで `SKILL.md` の概要と本文を確認可能
- 公開 skill の発想をローカル向けに落とし込める

### 6. MCP ツール生態系
- MCP ツール接続
- 人格 / 部門ごとにツール利用境界を統制
- 雑多な統合ではなく、運用可能な能力層として扱う

### 7. 私有部門と私有人格
- 主設定とは別に、AI 固有の組織空間を持てる
- 私有部門、私有人格、私有 skill をワークスペースから実行時へ反映
- 主助理と私有組織を並行進化させられる

### 8. 効率的で低コストな記憶
- 主動回憶
- 記憶分類
- 私有記憶
- 長対話の自動アーカイブ
- 文脈整理と圧縮
- バックエンド主導の文脈使用率計算

無限に文脈を詰め込むことが目的ではありません。  
AI を長く生かしつつ、コストを抑えることが目的です。

## 典型的なワークフロー

### ちょっとした質問
1. ホットキーでチャットを召喚
2. 質問、貼り付け、スクリーンショット
3. AI が回答またはツール実行
4. 閉じて作業に戻る

### 長期タスク
1. AI がタスクを作成
2. 目標時刻、周期、継続目標を設定
3. システムが追跡と通知を行う
4. 状態を段階的に更新し続ける

### 部門協働
1. 主助理が仕事を受け取る
2. 部門 / 工具人へ委任
3. 下位が独立実行
4. 結果が上へ戻る
5. 主助理が最終応答する

### ワークスペースの進化
1. AI が skill、私有人格、私有部門を管理
2. refresh で実行時に反映
3. 組織が現実の仕事とともに進化

## 誰に向いているか

- LLM を本気でデスクトップ作業に埋め込みたい人
- 一問一答では足りない人
- 長期タスクを AI に進めさせたい人
- AI に組織、委任、記憶を持たせたい人
- 他人の製品ルールではなく、自分の AI システムを作りたい人

## クイックスタート

1. アプリを起動
2. システムトレイから Config を開く
3. LLM API 設定を追加して保存
4. 主助理部門のモデルと委任人格を選ぶ
5. 召喚ホットキーを設定
6. チャットウィンドウから作業を開始
7. Tasks、Departments、Skills、MCP でシステムを広げる

## すでに実装されている主要要素

- デスクトップ AI オーバーレイ
- 自動アーカイブ
- 統一ツール呼び出しパイプライン
- タスクシステムとタスクボード
- 部門委任 / 工具人ワークフロー
- 私有組織ワークスペースの refresh
- preset / custom skill workspace
- MCP 統合と能力統制
- 文脈整理
- バックエンド主導の文脈使用率計算

## プライバシーとデータ

- API キーは既定でローカル保存
- 会話、アーカイブ、タスク、記憶、ワークスペースデータは既定でローカル保存
- 自分で管理、書き出し、削除できます
