# 远程IM渠道接入计划

## 做什么

让飞书、钉钉、NapCat(QQ) 的消息能进入现有聊天流程，让助理的回复能转发回去。

## 现有流程（不需要改的部分）

消息入队 → 写入历史 → 决定是否激活助理 → 助理流式回复 → 前端展示

这套全部现成。附件处理（图片、音频）也现成。

## 需要新增的东西

### 1. 协议适配层

每个平台一个适配器，职责只有两件事：

- **收消息**：连接平台，监听消息事件，把平台消息转换成系统内部的 ChatMessage 格式，调用现有的 `enqueue_chat_event` 入队
- **发消息**：把系统内部的 ChatMessage 转换成平台格式，调用平台 API 发出去

三个平台的连接方式：

| 平台 | 连接方式 | 方向 |
|------|----------|------|
| 飞书 | WebSocket 长连接 | 应用 → 飞书服务器 |
| 钉钉 | Stream 模式 | 应用 → 钉钉服务器 |
| NapCat | 反向 WebSocket | NapCat → 应用（本地监听） |

三个平台都不需要公网 IP。

附件收发方面，各平台有差异（飞书需要先上传拿 key、钉钉需要 media_id、NapCat 直接发路径），适配器内部处理，对外统一。

### 2. 渠道配置

用户需要能在设置页添加、编辑、删除渠道。每个渠道的配置：

- 平台类型（飞书 / 钉钉 / NapCat）
- 连接凭证（各平台不同：app_id/secret、client_id/secret、ws地址/端口）
- 是否启用
- 是否激活助理（消息进来后要不要触发助理回复）
- 是否接收附件
- 是否允许助理发送文件到 IM

### 3. 出站转发

助理回复完成后，需要一个钩子决定要不要把回复转发到 IM。

每个联系人（一个 QQ 群、一个飞书私聊）有自己的回复策略：

| 策略 | 含义 | 例子 |
|------|------|------|
| 不回复 | 只收消息，不转发助理回复 | 智能家居设备上报 |
| 始终回复 | 助理每次回复都转发 | 管理员 |
| 有消息才回复 | 对方说过话才转发 | 普通用户 |

判断机制：每个联系人记一个标记，收到消息时置 true，转发成功后置 false。

### 4. 消息来源标识

远程消息进入系统后，需要记录它来自哪个平台、哪个用户/群、发送者昵称和头像。前端展示时用这些信息显示消息来源，不要和系统内部人格混淆。

## 需要改动的东西

### 1. ChatEventSource 扩展

现有的消息来源枚举（User / Task / Delegate / System）需要新增一个远程 IM 来源，携带渠道和发送者信息。

### 2. 助理回复完成后的钩子

现有流程中助理流式回复完成后，需要加一个出站检查点：遍历所有启用的渠道，按联系人策略决定是否转发。

### 3. 设置页新增渠道管理入口

前端设置页需要新增一个区域，管理远程 IM 渠道的增删改查和连接状态展示。

### 4. AppData 扩展

配置持久化中需要存储渠道列表和联系人列表。

## 完成后的期望流程

### 入站（IM → 系统）

1. 用户在飞书群里发了一条消息 "帮我看看这个报表" 并附带一张截图
2. 飞书适配器收到消息事件
3. 适配器把文本转成 ChatMessage，把截图下载下来转成系统的图片附件格式
4. 适配器调用 `enqueue_chat_event`，消息进入现有队列
5. 现有流程接管：出队 → 写入历史 → 前端立刻看到这条来自飞书的消息（带平台图标、发送者昵称和头像）
6. 如果渠道配置了激活助理，助理开始处理并流式回复

### 出站（系统 → IM）

1. 助理流式回复完成，生成一条完整的助理消息
2. 出站钩子触发，遍历所有联系人
3. 这个飞书群的联系人策略是"有消息才回复"，且刚才确实收到了新消息
4. 把助理消息转换成飞书格式，通过飞书 API 发送到那个群
5. 清除该联系人的新消息标记

### 多联系人场景

1. QQ群A（on_message）的用户发了消息
2. 飞书群B（on_message）的用户也发了消息
3. 钉钉用户C（always）没说话
4. QQ设备D（none）上报了温度数据
5. 助理回复完成
6. 转发给：A、B、C（C 始终收到）
7. 不转发给：D（策略为不回复）

### 渠道管理

1. 用户打开设置页，看到"远程IM渠道"区域
2. 点击"添加渠道"，选择飞书，填入 App ID 和 App Secret
3. 点击保存，系统建立 WebSocket 连接
4. 连接成功后，渠道状态显示绿色"已连接"
5. 飞书群里有人说话时，消息自动出现在 APP 聊天窗口中
