# apply_patch 工具接入计划

## 目标

在现有内置工具体系中新增 `apply_patch`，支持通过结构化补丁对工作区文件进行增删改与重命名，并保证路径边界与审批行为可控。

## 需求范围

1. 工具形态
   - 新增独立内置工具 `apply_patch`。
   - 与 `exec` 分离，避免把文件编辑行为混在 shell 执行中。

2. 补丁语法
   - 支持 Codex 风格补丁语法：
   - `*** Begin Patch` / `*** End Patch`
   - `*** Add File:` / `*** Delete File:` / `*** Update File:`
   - `*** Move to:`
   - `@@` hunk 与 `+/-/ ` 行前缀

3. 能力要求
   - 支持文件新增、删除、更新、重命名。
   - 返回结构化执行结果（是否成功、变更项、错误信息）。

4. 路径与边界安全
   - 仅允许在当前会话已授权工作区内操作。
   - 路径越界（含 `../` 逃逸、越界重命名）必须拒绝。

5. 安全判定规则（三态）
   - 保留 `AutoApprove / AskUser / Reject` 三态。
   - 规则调整：
   - 当目标位于 LLM 默认工作区时，直接免审批（自动通过）。
   - 当目标位于用户选择的工具工作区时，才启用三态判定（可能询问用户或拒绝）。

6. 配置与运行时接入
   - 工具需要进入默认工具配置列表，并可在设置中开关。
   - 需要接入 runtime tool assembly 与工具状态检查链路。
   - 需要遵守部门工具可用性限制。

7. 日志与可观测性
   - 日志文案使用中文。
   - 输出任务名、触发条件、关键计数、耗时、失败原因。

8. 验收标准
   - 模型可调用 `apply_patch` 完成增删改移。
   - 默认工作区修改无需审批。
   - 用户工具区修改按三态规则运行。
   - 越界路径被拒绝且返回清晰原因。
   - 工具在配置页可控且状态检查正常。
