# 聊天代码块切换 Shiki 模式计划

## 背景

当前聊天代码块走 `markstream-vue + stream-monaco` 路线，在本项目的 Vite dev 环境下频繁触发 Monaco worker、依赖预构建与 sourcemap 噪音问题，导致白屏和启动不稳定。

`markstream-vue` 官方文档已提供更轻量的 Shiki 路线：安装 `shiki + stream-markdown` 后，通过 `setCustomComponents` 覆盖 `code_block` 节点为 `MarkdownCodeBlockNode`。

## 目标

- 聊天窗口代码块改走官方 Shiki 模式
- 代码块固定暗色主题，不再跟随应用亮暗主题切换
- 移除聊天入口对 Monaco worker 的依赖
- 收口 Vite 中仅为 Monaco 服务的配置，降低 dev 启动风险

## 实施步骤

1. 新增聊天专用的 Shiki 代码块包装组件，固定暗色显示。
2. 在聊天 Markdown 渲染链中用 `setCustomComponents('chat-markstream', { code_block: ... })` 覆盖官方代码块节点。
3. 移除 `main-chat.ts` 中的 Monaco worker 初始化。
4. 清理 `vite.config.ts` 中仅 Monaco 使用的 alias / optimizeDeps 配置。
5. 如无其他引用，移除 `stream-monaco` 依赖，避免继续拉入 `monaco-editor`。
6. 运行前端构建验证，确认聊天包可正常打包。

## 风险与回退

- 风险：流式代码块在未闭合阶段的表现可能与 Monaco 有差异。
- 风险：若 `markstream-vue` 当前版本的 `MarkdownCodeBlockNode` 在本项目样式体系下有兼容问题，需要额外补样式。
- 回退：恢复 `setCustomComponents` 覆盖前的默认 `CodeBlockNode`，并恢复 `main-chat.ts` / `vite.config.ts` 中的 Monaco 配置。
