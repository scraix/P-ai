# 聊天即时督工任务计划

## 背景

当前系统已经有两套相关能力：

1. 会话内 `todo`
用于追踪当前多步骤执行，但它更偏“步骤板”，不具备强提醒和跨轮督促语义。

2. 长期 `task`
具备触发、调度和持续提醒能力，但入口主要在任务面板和模型工具层，缺少“聊天中一键发起督工”的轻交互。

用户希望在聊天工具栏直接发起一个“督工任务”：

- 点击按钮后弹出简易卡片
- 填写：
  - 持续时间：`1 小时 ~ 24 小时`
  - 督工间隔：用户可主动选择
  - 目标
  - 原因
  - 待办
- 提交后立即生成一个立即开始的限时循环任务
- 在任务有效期内，系统持续督促 LLM 不要每做一步就停下来问用户，而是尽量连续推进；只有在任务真正完成、无法继续、或需要用户决策时，才主动停下

## 目标

新增一个“聊天即时督工任务”轻入口，复用现有 `task` 调度机制，不引入新的任务存储模型。

实现后用户可以在聊天窗口直接创建“限时督工任务”，让系统在指定时间窗口内强化执行连续性，减少 LLM 频繁停下来确认的低效行为。

## 非目标

- 不新增独立数据库表
- 不新增第二套“督工任务”后端模型
- 不改造现有任务面板为复杂工作流系统
- 不把所有对话都默认变成强督工模式
- 不强制跳过真正必要的用户确认和高风险决策

## 用户交互方案

### 工具栏入口

在聊天工具栏新增一个“督工”按钮：

- 位置：聊天区底部工具栏 `ChatWorkspaceToolbar.vue`
- 形态：小按钮，风格与现有工具栏一致
- 状态：
  - 空闲时可点击创建
  - 若当前会话已有活跃中的督工任务，则按钮显示激活态，并支持再次打开查看/覆盖

### 弹出卡片

点击后弹出一个简易创建卡片或弹层，字段仅保留：

- 持续时间：下拉选择 `1 ~ 24 小时`
- 督工间隔：下拉选择固定分钟档位，默认 `1 分钟`
- 目标：必填
- 原因：可选但建议填写
- 待办：必填

交互约束：

- 默认时长可设为 `4 小时`
- 督工间隔建议提供少量固定选项，例如 `1 / 2 / 5 / 10 / 15 / 30 分钟`
- 提交按钮文案建议为“开始督工”
- 关闭时若表单已改动，提示是否放弃

### 创建后的行为

提交后直接创建一个立即开始的限时循环任务：

- `runAtLocal = 当前本地时间`
- `endAtLocal = 当前本地时间 + N 小时`
- `everyMinutes = 用户选择的督工间隔`
- `goal / why / todo` 直接来自表单
- `conversationId = 当前聊天会话`
- `targetScope = desktop`

创建成功后：

- 前端给出轻提示，例如“已开始督工 X 小时”
- 当前会话进入“督工激活”状态
- 后续督工完全复用现有 task 调度器，不额外注入新的会话级系统提示词

## 后端方案

### 1. 复用现有 task_create_task

不新增新命令，优先复用已有：

- `task_create_task`

前端直接调用现有 Tauri 命令创建任务，只是由聊天侧包装成“即时督工任务”的预设输入。

### 2. 直接复用现有 task 调度器

这里不新增“督工模式”后端状态，也不在聊天主提示词里额外附加规则。

直接复用现有行为：

- 调度器每 `30s` 扫描一次到期任务
- 该限时任务会按 `everyMinutes = 用户选择值` 持续触发
- 每次触发时，现有 `task_trigger` 已经会向聊天线程注入隐藏任务提示，督促继续推进当前任务

也就是说，这个需求本质上是“聊天侧新增一个轻入口来创建适合督工的 task”，而不是“再造一层新的督工 prompt 体系”。

### 3. 不新增新的 task 工具动作

本期不扩展模型侧 `task` 工具协议。

原因：

- 用户发起督工任务是前端显式交互，不需要模型自己创建“督工模式”
- 减少工具协议扩张，降低回归面

## 前端方案

### 1. 在聊天工具栏增加督工按钮

涉及文件：

- `src/features/chat/components/ChatWorkspaceToolbar.vue`
- `src/features/chat/views/ChatView.vue`
- `src/UnifiedWindowApp.vue`

需要补的能力：

- 按钮点击事件
- 当前会话是否存在活跃督工任务的展示态
- 成功创建后的局部状态刷新

### 2. 新增轻量弹窗组件

建议新增一个聊天侧对话框组件，例如：

- `src/features/chat/components/dialogs/ChatSupervisionTaskDialog.vue`

职责：

- 维护本地表单
- 选择持续时间
- 选择督工间隔
- 校验必填项
- 生成 `task_create_task` 入参
- 提交成功后关闭弹窗并触发刷新

### 3. 会话级活跃督工状态

前端需要能知道“当前会话是否有活跃督工任务”，建议两种实现方式择一：

1. 最小改动方案
每次打开弹窗或切换会话时，调用一次 `task_list_tasks`，前端本地筛出当前会话的活跃督工任务。

2. 更稳方案
新增一个轻量查询命令，只返回当前会话活跃督工任务摘要。

本期建议优先用方案 1，改动最小。

## 数据约定

为避免引入新字段，本期先用“约定式识别”来标记督工任务：

- `conversationId = 当前会话`
- `targetScope = desktop`
- `everyMinutes` 为用户选定的督工频率
- `goal / why / todo` 为用户填写内容
- 并在 `why` 或 `goal` 前加一个稳定前缀标记，例如：
  - `督工任务：...`

更稳妥的做法是后续为 task 增加 `kind` 字段，但本期先不扩库。

## 风险与注意点

### 1. 督工不能覆盖安全边界

即使在督工模式下，也不能绕过：

- 高风险确认
- 不可逆操作确认
- 权限不足或信息不足时的必要停顿

### 2. 不能把长期任务和会话督工混淆

这个入口创建的是“立即开始的限时循环任务”，虽然复用了 `task`，但语义上是聊天中的执行督促，不应误导成长期计划管理。

### 3. 会话切换后的状态同步

如果用户切到别的会话，再切回来，按钮激活态都必须正确恢复。

### 4. 任务到期后的督工结束

任务到期后，循环触发自然停止，不需要额外收口聊天系统提示词。

## 实施步骤

1. 前端工具栏增加“督工”按钮与激活态展示
2. 新增简易督工任务弹窗，提交时调用 `task_create_task`
3. 创建时生成 `runAtLocal=现在 + everyMinutes=用户选择值 + endAtLocal=持续时间结束` 的 task
4. 在聊天会话切换、任务创建后刷新当前会话督工状态
5. 补测试：
   - 前端：表单校验与 1~24 小时时长转换
   - 前端：间隔选项与默认值是否正确
   - 前端：创建请求是否生成 `runAtLocal + everyMinutes=用户选择值 + endAtLocal`
   - 后端沿用现有 task 调度测试，不重复新增 prompt 规则测试

## 验收标准

- 用户可在聊天工具栏点击“督工”打开简易创建卡片
- 可选择 `1 ~ 24 小时`
- `目标 / 待办` 为空时不可提交
- 提交后立即创建绑定当前会话的限时循环 task
- 任务有效期内，系统依靠现有 task 调度器持续督工
- 任务到期或完成后，督工自然停止
- 不影响现有 `todo`、`task`、`delegate`、普通聊天发送与任务面板功能
