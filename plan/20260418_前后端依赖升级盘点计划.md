# 20260418_前后端依赖升级盘点计划

## 一、文档目的

这份文档用于记录当前仓库前后端依赖的落后情况，作为后续依赖升级的盘点依据。

本次盘点不直接执行升级，只回答三个问题：

1. 前端哪些依赖已经落后
2. 后端哪些依赖已经落后
3. `genai` 当前是否也存在更新空间

## 二、盘点口径

本次盘点采用以下口径：

- 前端：
  - 以 `pnpm outdated --format json`
  - 以及当前 `pnpm list` 实际安装版本为准
- 后端：
  - 以 `Cargo.toml` / `Cargo.lock` 当前声明与锁定版本为准
  - 用 `cargo search ... --registry crates-io` 查询 crates.io 当前最新公开版本
- `genai`：
  - 因为当前使用的是 git 依赖，不走 crates.io 版本号
  - 单独记录当前固定 rev 与上游 HEAD 的差异

## 五、genai 当前状态

`genai` 当前不是 crates.io 版本依赖，而是固定 git rev：

- 当前固定 rev：`d40abb2583f2aacdda0a51b79b164965ebda03a9`
- 当前上游 HEAD：`13a7348bba1538d45b808a9b0feb1090a4f650b4`

另外，crates.io 上公开版本当前显示为：

- `genai = 0.6.0-beta.17`

因此结论是：

1. `genai` 确实不是最新
2. 但它当前属于 git 跟踪模式，不适合直接和 crates.io 版本做一一对照
3. 如果后续要升级，应该单独做一次：
   - git rev 更新评估
   - API break 评估
   - 流式与工具调用链兼容评估

## 六、升级风险分层

### 1. 低风险补丁 / 小版本更新

这类通常可以优先考虑：

- `tauri 2.10.2 -> 2.10.3`
- `tauri-build 2.5.5 -> 2.5.6`
- `tauri-plugin-dialog 2.6.0 -> 2.7.0`
- `tauri-plugin-updater 2.10.0 -> 2.10.1`
- `tauri-plugin-single-instance 2.4.0 -> 2.4.1`
- `@tauri-apps/cli 2.10.0 -> 2.10.1`
- `vue 3.5.28 -> 3.5.32`
- `tailwindcss 4.2.1 -> 4.2.2`
- `postcss 8.5.6 -> 8.5.10`
- `stream-markdown 0.0.14 -> 0.0.15`
- `katex 0.16.33 -> 0.16.45`
- `captis 0.6 -> 0.6.1`

### 2. 中风险升级

这类可能涉及 API、类型或行为变化，需要单独验证：

- `async-openai 0.33.0 -> 0.34.0`
- `tokio 1.48.0 -> 1.52.1`
- `tokio-tungstenite 0.26 -> 0.29.0`
- `pdf_oxide 0.3.17 -> 0.3.33`
- `xcap 0.8 -> 0.9.4`
- `windows-sys 0.59 -> 0.61.2`
- `mermaid 11.12.3 -> 11.14.0`
- `markstream-vue 0.0.9-beta.1 -> 0.0.13-beta.2`

### 3. 高风险大版本升级

这类不建议和小修小补混在一起，应单独立项：

- `vite 6 -> 8`
- `@vitejs/plugin-vue 5 -> 6`
- `typescript 5 -> 6`
- `vue-i18n 10 -> 11`
- `shiki 3 -> 4`
- `lucide-vue-next 0.x -> 1.0`
- `cropperjs 1 -> 2`
- `rmcp 0.16 -> 1.5`
- `reqwest 0.11 -> 0.13`
- `rusqlite 0.32 -> 0.39`
- `tantivy 0.22 -> 0.26`
- `zip 2.x -> 8.x`
- `genai` git rev 升级

## 七、当前结论

当前仓库的依赖状态可以概括为：

1. 前端有一批可升级项
2. 后端也确实有不少已落后依赖，不只是前端
3. Tauri 主链整体不算非常落后，但仍有一组补丁版本可以追平
4. 后端真正跨度大的落后项集中在：
   - `rmcp`
   - `reqwest`
   - `rusqlite`
   - `tantivy`
   - `zip`
   - `genai`
5. `genai` 不是没更新，而是当前采用了固定 git rev，确实已经落后于上游 HEAD

## 八、后续建议

如果后续要动手升级，建议分三批：

1. 第一批：
   - 仅升级 Tauri 补丁版本和低风险前端补丁版本
2. 第二批：
   - 升级中风险依赖，逐项回归验证聊天、MCP、窗口与截图能力
3. 第三批：
   - 单独处理高风险大版本依赖和 `genai` / `rmcp` 这类协议级依赖

这份文档当前只做盘点，不包含实际升级执行。
