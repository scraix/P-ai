# 20260421_消息多选分享图片与HTML计划

## 一、目标

为当前已实现的消息多选模式补齐两种真实可用的分享能力：

1. 分享图片
2. 分享 HTML

其中：

- 分享图片用于生成一张可直接分发的长图
- 分享 HTML 用于生成一个独立可打开的静态历史页文件

这轮不做“分享到外部平台”的直接分发协议，只完成：

- 生成分享产物
- 让用户保存到本地
- 保证产物可以独立查看与转发

## 二、需求收口

### 1. 分享图片

用户在聊天窗口进入多选模式后：

- 勾选多条消息
- 点击 `分享`
- 选择“分享图片”
- 生成一张包含所选消息的长图
- 保存到本地

目标效果：

- 视觉上接近当前聊天记录
- 适合截图式传播

### 2. 分享 HTML

用户在聊天窗口进入多选模式后：

- 勾选多条消息
- 点击 `分享`
- 选择“分享 HTML”
- 生成一个独立的 `.html` 文件
- 文件可在浏览器中直接打开

这个 HTML 不依赖应用运行时，不请求后端接口，不依赖本地路由，而是：

- 一个单文件静态历史页
- 内联所需样式
- 用于本地打开、归档、转发

## 三、实现原则

### 1. 不导出整页应用

这轮的 HTML 分享不是导出整套前端页面，也不是打包聊天窗口运行态，而是：

- 从已选消息生成一份独立的分享数据
- 套入固定模板
- 输出纯静态 HTML

也就是说，导出的只是“历史页结果”，不是“应用页面快照”。

### 2. 优先复用现有消息结构

多选消息当前已经能用于：

- 复制
- 派生
- 投送

所以这轮应尽量复用现有“已选消息提取”口径，避免再造一套消息选择规则。

### 3. 先支持核心内容，复杂富媒体可降级

首版重点支持：

- 文本
- 代码块
- 图片
- 基本说话人标签
- 时间

可以接受的降级：

- 工具消息使用简化样式
- 音频显示为占位说明
- 附件先显示文件名或提示，不强求完整预览

## 四、分享图片方案

### 1. 输出形式

生成一张长图文件，例如：

- `share-20260421-143000.png`

### 2. 生成方式

建议走前端渲染后截图：

- 单独准备一个“分享渲染容器”
- 使用与聊天消息相同或近似的展示组件
- 渲染选中消息
- 再导出为图片

不建议在后端用字符串拼图片，因为：

- 当前消息布局本来就在前端
- 图片样式和代码块排版前端更成熟
- 前端更容易保证和聊天窗口视觉一致

### 3. 图片内容

首版包含：

- 顶部标题（如“PAI 对话分享”）
- 导出时间
- 已选消息内容

不强制包含：

- 输入框
- 工具栏
- 窗口壳

## 五、分享 HTML 方案

### 1. 输出形式

生成一个独立 `.html` 文件，例如：

- `share-20260421-143000.html`

### 2. 页面结构

建议固定成：

- 页面标题
- 导出时间
- 会话基础信息（可选）
- 消息列表

页面为纯静态：

- 不依赖 JS 应用初始化
- 不依赖接口
- 尽量不依赖外链资源

### 3. 样式策略

建议内联基础 CSS：

- 页面宽度
- 消息卡片
- 说话人
- 时间
- 代码块
- 图片最大宽度

这样导出的 HTML 文件双击即可查看，不受应用环境影响。

### 4. 内容边界

HTML 分享首版应支持：

- 文本消息
- 代码块
- 图片
- 助理/用户/系统消息基础标签

可接受的首版降级：

- 工具消息简化成文本块
- 音频/附件显示说明文字

## 六、前端交互

### 1. 多选操作条

当前多选操作条已有：

- 派生
- 复制
- 分享
- 投送
- 取消

这轮 `分享` 点击后不再提示“暂不支持”，而是打开一个分享方式弹窗。

### 2. 分享方式弹窗

弹窗中至少提供两个按钮：

- `分享图片`
- `分享 HTML`

可选补充：

- 简短说明
- 取消按钮

### 3. 保存行为

点击任一分享方式后：

- 生成对应产物
- 弹出保存文件对话框
- 用户选择保存位置

## 七、后端职责

### 1. HTML 文件写出

如果最终保存逻辑仍统一走 Tauri 文件写出，则后端需要提供：

- 生成后的 HTML 文本保存

### 2. 图片文件保存

如果前端直接拿到 Blob / base64，则后端只负责：

- 文件保存

### 3. 不承担页面渲染

后端不负责：

- 拼装聊天 UI
- 生成完整图片布局

这些应留在前端。

## 八、验收口径

### 1. 分享图片

- 多选至少一条消息后，可点击分享
- 选择“分享图片”后能成功导出 PNG
- 导出的图片能看到所选消息内容

### 2. 分享 HTML

- 多选至少一条消息后，可点击分享
- 选择“分享 HTML”后能成功导出 `.html`
- 双击打开 HTML 后能正确看到消息内容
- 页面不依赖应用运行

### 3. 基础一致性

- 分享导出的消息顺序与选中顺序一致
- 说话人、时间、文本内容正确
- 代码块和图片能基本正确展示

## 九、暂不包含

这轮暂不包含：

- 直接分享到微信/QQ/联系人
- Markdown 分享
- PDF 导出
- 音频富预览
- 附件下载能力完整移植到 HTML
- 复杂主题切换

## 十、后续可扩展方向

后续如果继续扩展分享能力，可再考虑：

- HTML 主题选择
- Markdown 导出
- PDF 导出
- 归档窗口复用 HTML 分享模板
- 联系人会话的一键分享
