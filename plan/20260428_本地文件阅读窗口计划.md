# 本地文件阅读窗口计划

## 背景

当前聊天 Markdown 中的本地绝对路径链接已支持打开文件所在目录，但代码与 Markdown 文件只能跳转到目录。项目内已经有 `markstream-vue`、Shiki 代码块组件与 Markdown 渲染链路，可以复用为只读文件预览能力。

## 目标

- 本地文件链接指向代码或 Markdown 文件时，打开独立文件阅读窗口。
- 文件阅读窗口支持多标签页：重复打开同一文件聚焦已有标签，打开新文件追加标签。
- 代码文件使用现有 Shiki 代码块组件浏览。
- Markdown 文件使用现有 Markdown 渲染链路浏览。
- 非可预览文件保持原有“打开所在目录”行为。

## 现状入口

- 本地链接点击入口：`src/features/chat/views/ChatView.vue::handleAssistantLinkClick`。
- 当前目录打开命令：`src-tauri/src/features/system/commands/desktop_tools.rs::open_local_file_directory`。
- 多窗口入口参考：`chat.html`、`archives.html`、`vite.config.ts` 的多入口配置。
- 独立窗口创建参考：`src-tauri/src/features/system/windowing.rs` 的独立聊天窗口。
- Markdown/Shiki 复用：`markstream-vue`、`registerChatMarkstreamComponents()`、`ChatShikiCodeBlockNode.vue`。

## 实施方案

### 1. 新增窗口入口

- 新增 `file-reader.html`。
- 新增 `src/main-file-reader.ts`。
- 新增 `src/apps/file-reader/FileReaderApp.vue`。
- 在 `vite.config.ts` 中增加 `fileReader` 入口。

### 2. 后端窗口与读取命令

- 新增 `open_file_reader_window(path)`：
  - 固定复用一个 `file-reader` 窗口。
  - 窗口不存在时创建 `file-reader.html?path=...`。
  - 窗口已存在时通过事件把路径投递给窗口，并聚焦窗口。
- 新增 `read_file_reader_file(path)`：
  - 只读文本文件。
  - 校验路径存在且是文件。
  - 限制最大读取体积。
  - 返回 `{ path, name, extension, kind, content }`。

### 3. 前端链接分流

- 在 `handleAssistantLinkClick` 中对本地绝对路径按扩展名分流。
- Markdown：`.md`、`.markdown`、`.mdx`。
- 代码/文本：`.ts`、`.tsx`、`.js`、`.jsx`、`.vue`、`.rs`、`.py`、`.json`、`.toml`、`.yaml`、`.yml`、`.css`、`.scss`、`.html`、`.xml`、`.sql`、`.sh`、`.ps1` 等。
- 可读文件调用 `open_file_reader_window`。
- 其他文件继续调用 `open_local_file_directory`。

### 4. 阅读窗口多标签页

- 窗口内部维护 tabs：`path/title/kind/content/loading/error`。
- 首次从 URL query 读取路径。
- 监听后端事件追加或聚焦文件标签。
- 支持关闭标签、切换标签、刷新当前文件、在目录中显示。

### 5. 渲染策略

- Markdown 文件：`MarkdownRender` + `parseMarkdownToStructure` + `chat-markstream`。
- 代码文件：包成 fenced code block 后交给 `MarkdownRender`，复用 Shiki 代码块组件。
- 普通文本：按 `txt` 语言走代码块展示。

## 边界

- 只读预览，不做编辑器。
- 不预览二进制、大文件、Office/PDF。
- 不改聊天 Markdown 主渲染链路，只复用现有组件。
- 目录打开能力保持不变。
