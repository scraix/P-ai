# 全局工具 Schema 缓存计划

## 目标

- 聊天发送前只读取全局 tool schema 缓存。
- 除 reload / 启动刷新外，不在发送链路检查工具可用性，不建立 MCP 连接，不列 MCP 工具。
- 只有 LLM 实际调用工具时，才检查该工具运行时是否可用。
- 工具执行阶段检验超时，向模型返回工具不可用，并提示 reload；reload 期间检验超时只记录明确日志。

## 边界

- 不改模型 provider、工具循环语义、权限模型。
- 不推进 aisdk 迁移。
- 不恢复临时聊天细分日志。
- 本次只处理 tool schema 缓存与执行期可用性检查时机。

## 实施点

1. 全局缓存
   - 在模型运行时维护进程级 tool schema 缓存。
   - 缓存内容只包含 `ProviderToolDefinition` 与日志 manifest 需要的静态信息。
   - `assemble_runtime_tools` 仅读缓存并组装执行器壳，不进行 MCP attach / list_all_tools。

2. reload 刷新
   - `builtin_reload` 完成 MCP / skill 刷新后重建 schema 缓存。
   - reload 期间 MCP 工具检验使用超时保护；超时写日志：某个 server/tool 检验超时。

3. 执行期检查
   - `operate` / `read_file` 等 MCP 工具改为懒连接。
   - LLM 实际调用时才启动 MCP 子进程、握手、查询目标工具并执行。
   - 连接或列工具超时，返回工具不可用结果，不阻塞聊天发送链路。

4. 验证
   - `cargo check`。
   - 静态检查发送前路径不再调用 MCP attach / list_all_tools。
   - reload 路径保留 schema 刷新入口。
   - 执行期 MCP 工具仍走超时保护。
