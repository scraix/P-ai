# 聊天右侧委托状态栏计划

## 背景

当前委托已经有独立的委托会话、运行线程、持久化记录和归档窗口的“委托会话”页，但聊天右侧栏没有面向当前会话的委托概览。用户希望在当前会话中能快速看到委托是否仍在执行、最近一次工具调用是什么、总共调用了多少次工具、执行了多久，以及模型请求次数和 token K 数；点击详情时跳到归档窗口中对应的委托会话。

## 目标

1. 在聊天右侧栏新增“委托”标签页。
2. 只展示当前前台会话相关的委托摘要，不展示全局无关委托。
3. 执行中实时显示：
   - 最近一次工具调用名称；
   - 工具调用总次数；
   - 已执行时间；
   - 当前状态。
4. 完成后显示：
   - 委托模型请求次数；
   - token 用量，按 K 展示；
   - 最终状态；
   - 完成/失败时间。
5. 点击“查看详情”打开归档窗口，并定位到对应委托会话。

## 非目标

1. 不在聊天右侧栏完整渲染委托消息内容。
2. 不新增独立委托详情窗口。
3. 不在本次实现复杂的委托成本统计或价格换算。
4. 不改变委托调度、执行、归档的核心流程。

## 数据来源与缺口

### 已有能力

1. 委托运行线程：`DelegateRuntimeThread`，包含 `delegate_id`、`root_conversation_id`、`conversation`、`archived_at`。
2. 委托会话列表：`list_delegate_conversations`，归档窗口已使用。
3. 委托详情消息：`get_delegate_conversation_messages`。
4. 归档窗口已经支持 `viewMode === "delegate"` 与 `selectDelegateConversation`。
5. 工具状态事件已在普通聊天流式链路中发送，包括工具开始、完成、失败等状态。

### 需要补充

1. 后端增加“当前会话委托状态摘要”查询命令，例如 `list_conversation_delegate_statuses(conversationId)`。
2. 摘要结构建议包含：
   - `delegateId`
   - `conversationId`
   - `rootConversationId`
   - `title`
   - `status`
   - `startedAt`
   - `updatedAt`
   - `completedAt`
   - `elapsedMs`
   - `requestCount`
   - `toolCallCount`
   - `lastToolName`
   - `tokenCount`
3. 运行中工具统计需要在委托执行链路中累计：
   - 最近一次工具名；
   - 工具调用次数。
4. token 用量按会话压缩边界统计：每次调度回来都会把当前用量写入消息/会话状态；对每个压缩边界的最后一条可信用量求和，得到委托累计 token。没有可信边界用量时显示为 `--`，不要编造估算。

## 实现方案

### 后端

1. 扩展委托运行态摘要
   - 新增轻量运行统计结构，挂在委托运行线程或单独的 `AppState` 运行态 map。
   - 在委托工具调用事件发生时更新 `lastToolName` 和 `toolCallCount`。
   - 记录开始时间、更新时间、完成时间，用于计算 `elapsedMs`。

2. 新增查询命令
   - `list_conversation_delegate_statuses(input: { conversationId: string })`
   - 返回当前主会话相关的委托，包含运行中、最近完成、已归档但仍在最近列表中的委托。
   - 过滤条件优先使用 `root_conversation_id == conversationId`，兼容持久化会话中的 `root_conversation_id`。

3. 详情定位
   - 复用归档窗口。
   - 前端通过 localStorage 写入 focus request，内容包含：
     - `viewMode: "delegate"`
     - `conversationId: delegateConversationId`
   - 归档窗口启动/刷新后读取该请求并切到委托页。

### 前端

1. 扩展右侧栏
   - 在现有 `ToolReviewSidebar.vue` 的 tablist 新增“委托”标签。
   - 新增 props：`delegateStatuses`、`delegateStatusLoading`、`delegateStatusError`。
   - 新增事件：`refreshDelegateStatuses`、`openDelegateDetail`。

2. 展示形态
   - 每个委托一张紧凑条目。
   - 常态展示：标题、状态 badge、执行时间、请求次数、token K。
   - 执行中额外展示：最近工具、工具调用总数。
   - 空状态：显示“当前会话暂无委托”。

3. 刷新策略
   - 当前会话变化时立即刷新。
   - 委托运行中每 2 秒轮询一次；没有运行中委托时停止轮询。
   - 收到委托开工/停工信号时触发一次刷新。

4. 跳转详情
   - 点击“查看详情”：
     - 写入归档焦点请求；
     - 调用 `show_archives_window`；
     - 归档窗口切到“委托会话”并选中对应 conversation。

## 验证

1. 后端最小测试：
   - 给定多个委托会话，只返回当前 root conversation 的委托。
   - 运行中摘要能计算 elapsedMs。
   - 没有 token 可信值时 token 字段为空而不是估算。

2. 前端最小验证：
   - `pnpm typecheck`
   - 右侧栏三标签切换正常。
   - 无委托、运行中、完成、失败四种状态展示正常。
   - “查看详情”能打开归档窗口并选中委托会话。

## 风险

1. 工具调用统计如果只依赖前端流式事件，刷新后会丢失；因此应优先在后端运行态累计。
2. token 用量必须按压缩边界最后一条可信用量求和，不能把每条消息的当前用量直接累加，否则会重复计算。
3. 归档窗口当前 focus request 可能只支持 current 会话，需要扩展为 delegate 模式，避免影响已有“当前会话摘要”跳转。

## 已确认口径

1. token K 数：使用每个压缩边界最后一条写入的当前用量，把这些边界值相加。
2. “步”：指模型请求次数。

## 实施记录

1. 后端新增 `list_conversation_delegate_statuses` 命令，按 `rootConversationId` 返回当前会话的委托状态摘要。
2. token 统计按压缩边界取最后一次 `effectivePromptTokens`，并包含当前未压缩片段的最新用量用于运行中展示。
3. “步”按模型请求统计：工具调用轮次中的 assistant tool-call 事件计为请求，最终文本回复另计一次请求。
4. 工具调用统计复用已持久化工具历史，并在运行中合并当前 inflight completed tool history。
5. 前端右侧栏新增“委托”标签，点击“查看详情”会打开归档窗口并定位到对应委托会话。
