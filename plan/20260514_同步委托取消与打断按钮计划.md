# 同步委托取消与打断按钮计划

## 背景

同步委托通过 `tokio::spawn` 把子会话放到独立任务里执行，再由主工具调用等待结果。此前公共工具层存在默认 30 秒超时，导致主工具调用被打断后，子委托任务仍在后台继续执行，右侧委托卡片步数继续增加。

公共默认超时已移除，但还需要补上取消语义：同步委托等待层一旦被用户停止、工具中止或外层取消，必须级联中止子委托任务和它当前正在执行的工具。

## 目标

- `sync delegate` 被外层取消时，不留下孤儿委托。
- 委托状态卡片在 `running` 时提供“打断”按钮。
- 打断按钮调用后端命令，停止该委托的聊天执行、当前工具执行、后代委托，并清理运行态。
- 不删除委托历史，保留可查看的已产生消息和工具记录。

## 后端方案

- 在 `run_sync_delegate_on_child_task` 中增加取消守卫。
- 子委托线程创建后注册到运行态，取消守卫可根据 `delegate_id` 调用统一中止函数。
- 抽取 `abort_delegate_runtime_thread`，按 `delegate_id`：
- 中止 `inflight_chat_abort_handles`。
- 中止 `inflight_tool_abort_handles`。
- 递归中止后代同步委托。
- 清理已完成工具历史。
- 释放会话处理声明并把运行态设为 Idle。
- 从 active runtime 移到 recent runtime，状态更新为 failed 或 interrupted。

## 前端方案

- `ToolReviewSidebar` 的 running 委托卡片增加“打断”按钮。
- `useDelegateStatus` 新增 `abortDelegate(status)`。
- 调用新命令 `abort_delegate_conversation`。
- 成功后立即刷新状态列表。
