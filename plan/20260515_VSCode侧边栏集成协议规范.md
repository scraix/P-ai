# VS Code 侧边栏集成 — 功能与界面规范

## 架构

### 通道复用

不新开 WebSocket 服务，在现有 IDE 上下文桥（`ide_context.rs`）上新增 `/chat` 路径。

现有基础设施：
- 动态端口：`43129`→`43139`，被占用自动顺延
- 发现文件：`p-ai-ide-context-bridge.json`，写入临时目录
- 路径：`/ide-context`（已有）+ `/chat`（新增）

```
ide_context.rs (WebSocket server @ 43129-43139)
├── /ide-context  ← 已有：VS Code 向 PAI 推送工作区上下文
└── /chat         ← 新增：侧边栏聊天 JSON-RPC 接口
```

VS Code 扩展读取发现文件获取 URL，连到 `ws://127.0.0.1:{port}/chat`。

### 前端代码位置

所有侧边栏代码集中在 `src/features/sidebar/` 下，与 `chat/`、`config/` 同级：

```
src/features/sidebar/
├── extension/              ← VS Code 扩展（package.json + extension.js）
├── App.vue
├── main-sidebar.ts         ← Vue 独立入口
├── sidebar.html            ← Vite 构建入口
├── layouts/
│   └── SidebarLayout.vue   ← 双视图切换
├── composables/
│   └── use-ws-transport.ts ← WebSocket 版 ChatTransport
├── views/
│   ├── ConversationListView.vue
│   └── ChatViewWrapper.vue
├── assets/
│   └── sidebar-theme.css   ← --vscode-* → DaisyUI 变量映射
└── dist/                   ← 构建产物
```

### 主题适配

不重写组件样式，通过 `sidebar-theme.css` 将 VS Code WebView 注入的 `--vscode-*` CSS 变量映射到 DaisyUI 的颜色变量（`--b1`、`--bc`、`--p` 等），实现亮/暗主题自动跟随。

## 布局

VS Code 侧边栏宽度固定（～300px），高度随窗口。只有两种视图，通过顶部按钮切换：

### 视图 A：会话列表

```
┌─────────────────────────┐
│ [<] [+]  [⋮]           │  ← 返回按钮 / 新建 / 更多
├─────────────────────────┤
│ 📄 帮我解释这段代码     │
│    2026-05-15           │
├─────────────────────────┤
│ 📄 优化一下这个函数     │
│    2026-05-14           │
├─────────────────────────┤
│ 📄 重构 UserService     │
│    2026-05-13           │
├─────────────────────────┤
│ ...                     │
└─────────────────────────┘
```

- 顶部栏：左侧返回按钮（回到当前会话/关闭列表）、新建按钮、更多菜单
- 列表项：会话标题 + 时间，点按进入会话
- 无分页，滚动加载更多

### 视图 B：对话

```
┌─────────────────────────┐
│ [≡] 当前会话标题  [⋯]   │  ← 打开列表 / 更多操作
├─────────────────────────┤
│                         │
│ 用户消息                │
│ ─────────────────────   │
│ AI 回复（markdown）     │
│ 推理过程 ▼              │
│ 代码块 [Copy]           │
│                         │
│ ─────────────────────   │
│ AI 回复 2               │
│ 工具调用 [exec] ✓       │
│ ─────────────────────   │
│                         │
├─────────────────────────┤
│ [输入框...]      [发送] │
└─────────────────────────┘
```

- 顶部栏 [`≡`]：打开会话列表；会话标题：显示当前会话名
- 更多菜单 [`⋯`]：新建会话、删除当前会话
- 对话流：和现有对话窗一致，markdown 渲染、代码高亮、推理折叠、工具调用卡片
- 底部：输入框 + 发送按钮（和现有一致）

### 切换逻辑

```
会话列表 ←── [≡] / [<] ──→ 对话
```

- 点 [`≡`] 切换到会话列表
- 点会话项切换回对话并加载该会话
- 点 [`<`] 回到上一个会话
- 新建会话自动进入对话视图并清空输入框

## 整体行为

- 侧边栏关闭/展开时保留当前会话状态
- 初次打开（PAI 未运行）时自动启动 PAI `--serve` 模式
- 断开连接时显示“PAI 未运行”提示，提供重连按钮
- 连接恢复后自动刷新当前会话
- 深色/浅色主题跟随 VS Code 设置

- MCP 管理面板
