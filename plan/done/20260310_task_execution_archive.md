# 20260310 Task 执行归档

## 1. 原始目标
- 将原先偏 schedule 的能力重新定义为 task。
- 让任务成为完整的任务系统，而不是单纯的定时器。
- 让 LLM 像正在推进任务的玩家一样，持续维护任务状态、目标、流程与待办。
- 让“当前追踪任务”自动出现在任务提示板中，持续驱动后续执行。

## 2. 最终落地结果
- 已新增完整的 task 模块，包含领域模型、存储、命令入口与调度器。
- 已提供任务工具动作：
  - `create`
  - `update`
  - `complete`
  - `list`
  - `get`
- 已落地任务提示板与任务触发卡片，LLM 会持续看到当前追踪任务。
- 已新增预设 Skill：`task-guide`，专门指导 LLM 如何写任务、推进任务、完成任务。
- 已新增前端任务面板，用于查看任务列表与任务详情。

## 3. 最终任务模型
- 任务核心字段已围绕以下语义落地：
  - `title`
  - `cause`
  - `goal`
  - `flow`
  - `todos`
  - `statusSummary`
  - `completionState`
  - `completionConclusion`
- 不允许删除任务。
- 当前追踪任务不允许由 LLM 手动控制，改由系统自动计算。

## 4. 最终触发器语义
- 最终没有保留 `trigger.kind` 这类额外标签。
- 对外工具与数据结构统一收口为：
  - `run_at`
  - `every_minutes`
  - `end_at`
- 语义如下：
  - 无 `run_at`：立即任务
  - 只有 `run_at`：一次性任务
  - `run_at + every_minutes + end_at`：有边界的周期任务
- 周期任务必须有起始时间与结束时间，避免无限悬空触发。

## 5. 当前追踪任务与提示板
- 当前追踪任务由系统自动判断，不暴露给 LLM 控制。
- 隐藏任务提示板会注入：
  - 当前本地时间
  - 当前追踪任务
  - 任务目标、起因、流程、状态摘要、待办
  - 任务的 `runAt / endAt / nextRunAt`
- 任务触发后，聊天里会生成结构化任务提醒卡片，而不是一段普通文本。

## 6. 时间格式策略
- 内部存储仍基于绝对时间。
- LLM 可见的任务时间统一改为本地 RFC3339 格式，并带时区偏移。
- 当前提示板与任务消息中，LLM 直接复制同样格式即可，不需要自行换算时区。

## 7. 前端与交互结果
- 已新增任务配置页签与任务详情面板。
- 任务触发消息改为任务中心人格发送，并按外来消息样式展示。
- 聊天界面可直接看到任务提醒卡片中的标题、目标、流程、状态与待办。

## 8. 调度与运行结果
- 调度器会检查到期任务，并在聊天繁忙时进入队列等待。
- 队列恢复、任务发送成功、任务发送失败等运行日志已补齐中文诊断信息。
- 当前实现优先保证“任务能被稳定触发并重新投递给 LLM”，后续若要继续强化队列策略，可再单独立项。
