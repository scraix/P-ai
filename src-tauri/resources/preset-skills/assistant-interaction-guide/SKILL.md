# Assistant Interaction Guide

## 核心目标

这个 skill 用来帮助用户理解一件事：

- 助理不只是聊天工具。
- 助理可以直接帮用户执行安装、配置、排查、整理和交付工作。
- 用户不需要先学会所有术语，只需要把目标说清楚，助理就可以带着用户往前走。

## 你要传达给用户的关键信息

当用户不知道怎么开口时，要优先让用户明白下面这些点：

1. 用户可以直接说目标，不必先想好完整方案。
2. 用户可以要求助理直接动手，而不只是解释。
3. 用户可以让助理帮忙安装基础环境。
4. 用户可以让助理帮忙安装 skill。
5. 用户可以让助理帮忙安装和配置 MCP。
6. 用户可以让助理帮忙安排部门、人格和工作流。
7. 如果用户表达不清，助理应该主动把问题缩成少量关键确认，而不是让用户自己摸索。

## 优先引导方式

如果用户很模糊，先用这种方式引导：

- 先告诉用户“你直接说目标就行，我可以帮你一步步做”。
- 再补一句“如果你不确定怎么描述，我也可以先问你最关键的几个问题”。
- 然后给出 3 到 5 个可直接照抄的示例。

避免：

- 一上来堆术语。
- 要求用户先理解 skill、MCP、部门这些概念再开始。
- 连续抛很多开放问题，让用户更慌。

## 用户可以直接这样说

### 1. 安装 Git

可以直接告诉用户这样说：

- 帮我安装 Git，并确认命令行可以用。
- 帮我检查这台机器有没有 Git，没有就安装。
- 帮我把 Git 配好，然后告诉我下一步怎么开始。

### 2. 安装 Node.js

可以直接告诉用户这样说：

- 帮我安装 Node.js，并检查 npm 是否正常。
- 帮我装一个适合当前项目的 Node 版本。
- 帮我把这个前端项目需要的 Node 环境配好。

### 3. 安装 Python 和 uv

可以直接告诉用户这样说：

- 帮我安装 Python，并确认命令行可以用。
- 帮我安装 uv，并帮我把这个 Python 项目环境建好。
- 帮我检查这台机器有没有 Python 和 uv，没有就直接装好。
- 帮我把这个仓库需要的 Python 环境部署好。

如果用户不知道 uv 是什么，就直接解释：

- uv 可以理解成更现代的 Python 包与虚拟环境工具。
- 用户不需要先会用，助理可以直接帮他安装、建环境、装依赖，再告诉他后续怎么继续。

### 4. 安装 skill

可以直接告诉用户这样说：

- 帮我找一个合适的 skill 并安装。
- 帮我把这个 skill 安装到当前工作区。
- 帮我整理一下现有 skill，看看缺什么。

如果用户自己都不知道 skill 是什么，就直接解释：

- skill 可以理解成“给助理追加一套专门能力说明和工作方法”的小模块。
- 用户不需要自己手写，助理可以代为安装、整理和创建。

### 5. 安装 MCP

可以直接告诉用户这样说：

- 帮我安装这个 MCP，并验证能不能连通。
- 帮我把当前工作区需要的 MCP 配起来。
- 帮我查一下这个 MCP 应该怎么装，然后直接配好。

如果用户不知道 MCP 是什么，就直接解释：

- MCP 可以理解成“让助理接入额外工具或数据源”的协议和运行方式。
- 用户不需要先懂细节，助理可以先帮他安装、配置，再解释当前接上了什么能力。

### 6. 安排部门

可以直接告诉用户这样说：

- 帮我安排几个部门，让助理分工合作。
- 帮我设计一个适合这个项目的部门结构。
- 帮我把主助理、执行助理、研究助理这些角色安排清楚。

如果用户不知道“部门”是什么意思，就直接解释：

- 部门就是把不同职责拆开，比如主助理负责对外沟通，某个部门负责代码，某个部门负责调研，某个部门负责整理资料。
- 这样做的目的是让长期工作更稳定，不是让用户多学概念。

## 用户提需求的推荐格式

当用户愿意多给一点信息时，引导他尽量补这几类内容：

1. 目标是什么。
2. 当前卡在哪里。
3. 项目或环境在哪。
4. 希望助理“解释”“操作”还是“两者都要”。
5. 有没有限制条件。

可以直接给用户这种句式：

- 目标：我要把这个项目跑起来。
- 当前情况：我还没装 Git 和 Node。
- 路径：项目在某个目录。
- 希望你做什么：直接帮我安装并检查。
- 限制：不要动无关项目。

## 当用户完全不会描述时

如果用户只会说：

- 我不知道怎么用你
- 你能帮我干嘛
- 我现在该怎么办
- 我不会装这些东西

你应该直接进入引导，而不是等用户自己组织语言。

推荐回应方式：

1. 先安抚：直接告诉用户“你不用先学会怎么说，给我一个目标就行”。
2. 再给选项：例如“你现在是想装环境、装 skill、装 MCP，还是想安排部门？”。
3. 再给示例：让用户直接复制一句最接近的话。

## 建议给用户的最小示例集

优先给这几句：

- 帮我安装 Git。
- 帮我安装 Node.js，并检查项目能不能跑。
- 帮我安装 Python 和 uv，并把环境配好。
- 帮我安装一个适合当前工作的 skill。
- 帮我安装并配置 MCP。
- 帮我安排部门，让助理能分工处理任务。
- 我不知道怎么描述，你先问我最关键的两个问题。

## 你自己的行为约束

当这个 skill 触发时，你应该：

1. 优先降低用户表达门槛。
2. 优先把“你可以直接让我做什么”说清楚。
3. 优先使用短句和可复制示例。
4. 不要把说明写成长篇概念教学。
5. 如果用户已经表达出明确目标，就直接进入执行，不要继续泛泛介绍。
6. 如果用户明显是在安装或部署环境，而终端可用，要主动询问是否需要你直接代为完成安装和部署。
7. 如果终端可用且用户同意，应优先实际完成环境部署，而不是只给教程。
