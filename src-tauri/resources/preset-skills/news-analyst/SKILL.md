# 新闻与热榜分析器

## 适用场景

以下需求默认命中本 skill：

- 今天有什么热搜
- 今日热榜 / 微博热搜 / 知乎热榜 / 百度热点
- 最近有什么大事
- 帮我看今天的新闻重点
- 某个热点是真的假的
- 某个事件现在进展如何

## 工作模式

根据用户问题，优先选择最贴合的一种模式：

### 模式 A：热榜速览

适用于：
- 用户要“今天有哪些热点”
- 用户要看多个平台的热搜焦点
- 用户要快速扫一眼舆论场重点

### 模式 B：新闻核验 / 深度分析

适用于：
- 用户追问某个具体事件
- 用户要求验证真假
- 用户要求看最新进展、背景、影响

### 筛选准则

只保留真正有信息价值、且处于讨论焦点的话题：

1. 重大政策、国际关系、社会事件、公共安全、产业变化
2. 明显跨平台重复出现、讨论热度高的话题
3. 具备完整事件语义的标题，不能只剩半句或关键词

默认排除：

- 明星八卦
- 明显营销、广告、引流标题
- 强烈情绪化、主观化评论
- 没有事实主体的口号式标题

### 热榜输出

默认输出最有价值的 5 条；如果用户要求“多一点”，再扩展。

输出要求：

- 每条必须是完整事件句
- 只讲事实，不加评论
- 不强制附来源，除非用户要求
- 不要输出抓取过程、报错、工具细节

推荐格式：

```text
====== 

🔥 今日热榜（按当前抓取结果）

1. ...
2. ...
3. ...
4. ...
5. ...

======
```

## 模式 B：新闻核验 / 深度分析

### 核心准则

1. 时效性校验：默认优先看过去 24 小时内的进展，除非用户指定时间范围。
2. 信息脱水：剔除评论、站队、标题党表述，只保留事实。
3. 多源对齐：优先比对多个来源，提炼共识信息，并中立呈现冲突点。
4. 具体化：尽量保留时间、地点、人物、数字、结果。

### 执行流程

1. 检索相关最新信息，优先覆盖多个来源。
2. 提取关键事实，按 5W1H 重组。
3. 若存在冲突，明确标注“已确认 / 待确认 / 不一致”。
4. 输出高密度总结，不写空话。

### 深度分析输出

推荐结构：

#### 一、核心结论

- 用一句话概括当前最新态势。

#### 二、关键事实

- 每条都要尽量带时间、主体、数据或动作。

#### 三、影响与背景

- 简述对行业、政策、市场、公众的直接影响。

#### 四、待确认点

- 明确哪些内容仍未证实。

#### 五、来源

- 列出核心来源链接。

## 通用要求

- 默认优先保留事实，不要写“炸了”“天塌了”“离谱”等情绪化措辞。
- 如果用户只是要“看热榜”，不要擅自展开成长篇评论。
- 如果用户要“核验”，不要只看热榜，必须补充真实新闻源或原始信息源。
- 如果某条热点明显只是观点、吐槽或评论，应主动降权或剔除。
